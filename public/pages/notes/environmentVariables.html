Environment Variables

What is environment variables?

Environment variables is a variable we can define outside of our code that can make different changes to our application.

Why do we use them?

Environment variables can have a lot of differnt use cases, but we primaraly use them at it makes our code flexable & to hide sencitive information in our application.

Here are some of the most common use cases we have used so far:

● API keys & secrets

● Database connections

● Port numbers

In node we will install a dependency called "cross-env" as environment variable are set differntly on different platforms.

This makes it easy for us as developers as we can have a single command without worrying about settings or using the environment variables properly for the platform we are on.

To install cross-env we will run this command:

$ npm install cross-env

Using Environment Variables in Node.js

In Node.js we use the process.env object to access environment variables

Here is an example from my project: 

"scripts": {
    "start-dev": "cross-env PORT=8080 NODE_ENV=DEV nodemon --ext js,html,css app.js",
    "start-prod": "cross-env PORT=80 node app.js"
  },
const PORT = process.env.PORT || 8080;

const server = app.listen(PORT, () => {
    console.log('Server started in port ', server.address().port);
});

Here we have two scripts. One of them if for development and the other is for production.

In the "start-dev" we start by writing cross-env as this will work for cross-platform.

PORT=8080 sets our port to 8080. In our app we can then use process.env.PORT to access the port we just set.

NODE_ENV=DEV tells the app it's running in dev mode.

We use nodemon instead of node as we want our app to hot reload when we make changes in js, html or css files!

The last thing is app.js where we tell nodemon where to run our app from.
