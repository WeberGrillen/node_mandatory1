Scope

Scope = Visibility

What is a Scope?

Scope determines the accessibility of a variable.

In JavaScript there is 3 kinds of scopes:

● Global scope

● Function scope

● Bloack scope

Global Scope

Variables declaied in the global scope is accessable everywhere in a JavaScript file.

Here is an example:

let name = "Frederik"; // global

function greet() {
    console.log(name); // able to access
}

Global variables has a Global Scope.

Function Scope

Variables declaied inside a function will not be accessable outside of a function

Here is an example:

function greet() {
    const name = "Frederik"; // function scope

    console.log(name); // able to access name as it's still in the function
}
greet();
console.log(name); // ReferenceError: name is not defined <- outside scope

Block Scope

Variables declaied inside a block will not be accessable outside the block

This only applies to let and const and is why we want to avoid using var

{
  const x = 2; // block scope
}
// x can NOT be used here <- outside scope
// ReferenceError: x is not defined

Why we DON'T ever use var

The reason to always want to avoid var is that var leaks to the next tier of scope.

So if we have a var in a block scope it will leak into a function scope, and a var in a function scope will leak into global scope which creates unpredicablility!

Example of block leaking into function scope:

function greet() {
    if (true) {
        var name = "Frederik"; // declared in block
    }
    console.log(name); // "Frederik" leaked into function!
}
function greet() {
    var name = "Frederik"; // declared in function
}

greet();
console.log(name); // "Frederik" leaked into global!
