Type Coercion

JavaScript secretly converts your values behind the scenes.

What is Type Coercion

Since JavaScript is losely typed it will try to convert data types to whatever it thinks it right.

This makes it easy for a beginner developer who just code something fast, but can quickly create a lot of problem down the road

A prime example would be multiplying a string "5" with a Number 5

console.log("5" * 5); // = 25 ( string + number ) -> number

As you can see it also changes the datatype from string to number

This is an effect we heavily want to advoid as it can make our code hard to understand and unpredicable

Use === instead of ==

When we want to compare two different primitives we always want to use strict equality (===) as this prevents unnecessary types of coercion

Example:

console.log("5" == 5); // = true
console.log("5" === 5); // = false
